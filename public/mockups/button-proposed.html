Proposed Button System

Two families: Accent (explore, navigate, tools) and Green (decide, commit, choose).

Pick the Accent Color

Accent Family — Explore / Navigate / Tool

CHANGE: --accent moves from #5b8def (blue) to #a78bfa (purple). All buttons using var(--accent) update automatically.
Solid Purple — Primary
.dl-execute, .scenario-proposal-confirm

Full-width primary actions. "Execute decisions", "Explore N areas." The most prominent purple button. Was blue.

Solid Purple — Confirm
.scenario-proposal-confirm

Confirms domain selection and kicks off exploration. Same solid purple as execute. Was blue.

Purple Icon — Send
.convo-send (large), .scenario-explore-send (small)

Chat send button and explore bar send. Icon-only. Was blue.

Mode Switcher
.mode-btn / .mode-btn-active

Active tab now purple. Inactive stays ghost. Was blue.

Tinted Purple — Tool Actions
.scenario-alloc-action-btn (all variants), .scenario-alloc-analyze-inline

All allocation tools now share the same tinted purple style. Undo and Duplicate were ghost/border-only, Analyze changes was yellow. Now unified. Loading spinner on analyze.

Purple Chip — Explore Prompts
.scenario-chip (unified)

All explore prompts are now one purple tint. No more blue vs green distinction. Subtle border, light fill on hover.

States: default, hover, loading (pulse), used (strikethrough)
Loading prompt... Already explored
Purple Chip — Loading / Used
.scenario-chip-loading / .scenario-chip-used

Loading pulses while AI responds. Used dims and strikes through. Unchanged behavior, just purple now.

Green Family — Decide / Commit / Choose

CHANGE: One green button for all commit actions. No more filled vs stroked, no warning/info variants. Green = "commit to this."
Green — All Commit Actions
.scenario-cta-btn, .scenario-comp-choose, .scenario-alloc-btn-decide

One style for every "commit" button: CTAs, Choose, Decide. Same tinted green with border. Full-width or auto-width depending on context. Was three separate styles with different fills/strokes.

States: default, hover, loading (spinner), disabled/done
Green — Loading / Done
.scenario-btn-loading / :disabled

Shared spinner and disabled treatment across all green buttons.

Unchanged

Ghost — Utility
.topbar-btn

Canvas utility buttons. Stays neutral ghost — these are ambient tools, not primary actions.

Design System Summary

Purple = Explore / Navigate / Tool
Solid — primary actions (execute, confirm, send)
Tinted — tool actions (undo, duplicate, analyze)
Chip — explore prompts
Green = Decide / Commit / Choose
One style — CTAs, choose, decide (all the same)
Neutral = Ambient
Ghost — utility buttons (fit, inactive modes)
