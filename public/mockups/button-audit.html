Button Audit

Every interactive button style in EE Graph Studio, with states and usage context.

Global / Shell

.topbar-btn
Topbar — top-right corner

Utility button for canvas actions. Currently only "Fit" (zoom-to-fit). Ghost style, no fill.

.mode-btn / .mode-btn-active
Topbar — mode switcher

Switches the active mode. Active tab gets solid accent fill. Only one active at a time.

.dl-execute
Decision Log — footer

Commits all queued decisions. Disabled when cart is empty. Full-width solid accent button.

States: enabled, disabled (opacity 0.4)
.convo-send
Conversation — input bar

Sends the user's message. Same as pressing Enter. Icon-only square button.

States: enabled, disabled (opacity 0.4)

Scenario — Explore Prompts

Which reports are highest risk? Compare interim candidates
.scenario-chip .scenario-chip-k / .scenario-chip-a
Explore bar — below each canvas card

Knowledge chips are blue — explore/understand. Action chips are green — decide/do. Clicking sends the prompt to the AI.

States: default, hover, loading (pulse), used (strikethrough + dim)
Loading... Already asked
.scenario-chip-loading / .scenario-chip-used
Explore bar — after click

Loading pulses while AI responds. Used state marks the prompt as consumed. Both disable pointer events.

.scenario-explore-send
Explore bar — custom input

Sends a freeform question from the explore bar's text input. Small purple accent, pairs with the input field.

Scenario — Call-to-Action Buttons

.scenario-cta-btn
Canvas cards — below card body

AI-generated action buttons. Three severity styles: approve (green, safe), warning (yellow, risky), info (blue, neutral). Full-width, prominent.

States: default, hover, loading (spinner), disabled, done
.scenario-btn-loading / :disabled
Canvas cards — after click

Loading state hides text and shows a spinner. Done state disables the button. The .scenario-btn-loading class works on any button.

Scenario — Option Selection

.scenario-comp-choose
Expanded option detail card — bottom

Selects a candidate/option from a comparison set. Green accent, matches "approve" CTA family. On click: dims siblings, marks this card as decided, sends choice to AI.

States: default, hover, chosen (disabled + "✓ Chosen")

.scenario-proposal-chip
Conversation — after initial analysis

Toggle-selectable domain chips. User picks which impact areas to explore. Selected state adds purple border and glow. Multi-select, then confirm.

States: default, hover, selected (purple glow)
.scenario-proposal-confirm
Conversation — below domain proposals

Confirms selected domains and kicks off exploration. Solid accent, full-width. Label updates with count.

Allocation — Actions

.alloc-action-btn + variants
Allocation card — action bar at bottom

Four allocation actions. Undo/Duplicate are ghost buttons (border only). Analyze is purple (triggers AI). Decide is green (commits scenario). Decide is disabled when analysis is stale.

.scenario-alloc-analyze-inline
AI Analysis header — when analysis is stale

Appears in the analysis panel header after user drags people. Yellow/warning style signals "your changes haven't been analyzed yet." Supports .scenario-btn-loading spinner.

Branching Mode

.branch-decide-btn
Comparison column — bottom of each option

Commits a branching decision. Solid green, same "go" energy as .scenario-comp-choose. Dims sibling columns on click.
